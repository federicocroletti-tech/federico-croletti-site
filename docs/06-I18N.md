# i18n - Internazionalizzazione

## Obiettivo
Preparare il sito per italiano e inglese, anche se la prima versione può partire solo in italiano.

## Lingue previste
- Italiano: `it`
- Inglese: `en`

## Strategia consigliata
Usare `ngx-translate` se si vuole gestire contenuti da file JSON in modo semplice e veloce.

Alternative:
- Angular i18n nativo: ottimo per build separate e contenuti più statici.
- ngx-translate: più pratico per cambio lingua runtime.

Per questo sito si consiglia `ngx-translate`.

## Struttura file
```txt
assets/i18n/it.json
assets/i18n/en.json
```

## Language switcher
Creare un componente:
- `LanguageSwitcherComponent`

Funzioni:
- mostra lingua corrente;
- permette cambio IT/EN;
- salva preferenza in `localStorage`;
- fallback su `it`.

## Best practice
- Non lasciare testi hardcoded nei componenti, tranne label tecniche non visibili.
- Usare chiavi ordinate per sezione.
- Evitare chiavi troppo generiche come `title1`, `text2`.
- Tradurre anche meta title e description se possibile.
- Preparare modelli dati con campi traducibili oppure usare chiavi i18n nei dati.

## Esempio dati progetto con i18n key
```ts
export interface ProjectItem {
  id: string;
  titleKey: string;
  descriptionKey: string;
  technologies: string[];
  githubUrl?: string;
}
```
